## Trigger Map

| Trigger Type | Specific Conditions |
|---|---|
| **Converts when** | Vendor provides detailed implementation timeline with milestones / Feature request aligns with published product roadmap / Support team demonstrates understanding of specific use case |
| **Abandons when** | Multiple software crashes occur during feature evaluation / Support responses are clearly automated without human review / Proposed features would disrupt existing workflow integrations |
| **Trusts when** | Vendor provides documented case studies from similar organizations / Regular product updates demonstrate ongoing platform investment / Support team provides detailed technical responses to complex questions |
| **Suspicious when** | Promised features consistently miss delivery dates / Support tickets are closed without resolution / New updates introduce bugs in previously stable features |

### Deal Breakers (Immediate Exit)

- 🚫 Applications that crash during peak productivity hours
- 🚫 Support teams that provide automated responses without human follow-up
- 🚫 Feature updates that break existing functionality without warning
